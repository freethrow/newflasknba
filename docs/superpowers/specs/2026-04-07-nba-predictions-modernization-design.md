# NBA Predictions — Modernization Design

**Date:** 2026-04-07  
**Status:** Approved  
**Approach:** Option A — strict sequential phases, fresh `src/` directory alongside legacy code

---

## 1. Context

Legacy Flask app for a Serbian friend-group NBA playoff prediction game. Tracks users, series (matchups), predictions, scores, and comments. The existing `nba.sqlite` database contains the full history and can be used as a model.

**Current stack:** Python 2 · Flask 1.0.2 · Flask-SQLAlchemy 2.3 · Flask-Admin 1.5 · Flask-Bootstrap4 · Flask-Script · SQLite

**Problems:** Python 2 syntax, broken itsdangerous import, Flask-Admin auto-CRUD (clunky for this use case), Bootstrap 4, committed `.pyc` files, no tests, no migrations, werkzeug password hashing.

## 4. Domain Model (preserved)

- **User** — `username`, `password_hash`, `is_admin`, `is_active`, `score`. `total` property sums `score_made` across predictions for the current season.
- **Series** — `home`, `away`, `open` (bool), `result` (e.g. `"4:2"`), `season` (string e.g. `"2026"`).
- **Prediction** — FK to `Series` + `User`, `predicted` string, `score_made` int. Unique per `(series_id, user_id)`.
- **Comment** — `body`, FK to `User`, `created`. No threading, no editing, no markdown.
- **AdminLog** — `who`, `when`, `action`, `target_id`, `before`, `after`. Append-only audit trail for every admin write.
- **Message** — `id`, `from_user_id` (FK User), `to_user_id` (FK User), `body` (Text), `created` (DateTime), `read` (bool, default False). Constraint: at least one of sender or recipient must be an admin — enforced at the route layer, not DB level.
- **`calculate_score(pred, actual)`** — extracted to `scoring.py`. Semantics preserved exactly: 15 pts for correct winner, +10/5/2 bonus by loser game count proximity, +1 if away team wins.

## 5. Phases

### Phase 0 — Repo hygiene
- Add `.gitignore` (excludes `*.pyc`, `__pycache__/`, `.venv/`, `instance/`, `.env`, `node_modules/`)
- Remove committed `.pyc` files
- Back up `nba.sqlite` → `nba.sqlite.bak`, then move to `instance/nba.sqlite`
- Delete `reqs.txt`
- Create feature branch `modernize`

### Phase 1 — Python 3 + Flask 3 port (no feature/UI changes)
- `requirements.txt` with: `flask>=3.1`, `flask-sqlalchemy>=3.1`, `flask-login>=0.6.3`, `flask-wtf>=1.2`, `flask-migrate>=4`, `argon2-cffi`, `python-dotenv`, `faker`
- Build `src/nba_predictions/` package with `create_app()` factory
- Convert Python 2 syntax: `print()`, remove `unicode()`, fix `is_authenticated` (property not method)
- Remove `itsdangerous.TimedJSONWebSignatureSerializer` import
- Replace Flask-Script `manage.py` with `flask --app src.nba_predictions` CLI + `@app.cli.command()` decorators
- Switch models to SQLAlchemy 2.0 typed style (`Mapped`, `mapped_column`)
- Add `season` column to `Series` (string, indexed); backfill existing rows after inspecting `nba.sqlite`
- Replace `forgery_py` with `Faker` in `flask generate-fake` CLI command
- Argon2 password hashing; on successful login detect legacy werkzeug hash by prefix and re-hash transparently
- Run `flask db init` → `flask db migrate` → `flask db upgrade` against a copy of `nba.sqlite`; verify all rows survive
- Smoke-test: register, login, leaderboard, submit prediction

**Exit criterion:** app runs on Python 3.12 + Flask 3.x, all existing pages functional, no UI changes yet.

### Phase 2 — Auth overhaul
- Argon2 passwords (already introduced in Phase 1; Phase 2 adds the full auth hardening)
- CSRF globally enforced: `CSRFProtect(app)`; all forms render `{{ form.csrf_token }}`
- `flask-limiter`: rate-limit `/auth/login` to 5 req/min per IP
- Password reset flow: `itsdangerous.URLSafeTimedSerializer` (1-hour expiry), email sent via Resend SDK wrapped in `mail.py`. Env vars: `RESEND_API_KEY`, `MAIL_FROM`
- Secure session cookies in prod config: `SESSION_COOKIE_SECURE=True`, `SESSION_COOKIE_HTTPONLY=True`, `SESSION_COOKIE_SAMESITE='Lax'`
- `admin_required` decorator using `current_user.is_authenticated` + `current_user.is_admin`
- `User.is_active` column; admins can disable accounts without deleting
- `flask user promote <username>` CLI command
- Tests: login, logout, register, password reset, admin-only route protection

**Exit criterion:** all auth flows work, login rate-limited, argon2 passwords, admin promotion via CLI.

### Phase 3 — Custom admin blueprint (replaces Flask-Admin)
- New blueprint `admin/` mounted at `/admin`; every route decorated with `@admin_required`
- **Dashboard** (`/admin`):
  - Series table: `home`, `away`, `result`, `open` toggle — inline edit via HTMX `hx-patch /admin/series/<id>`
  - Users table: username, score total, `is_admin`, `is_active` — promote/demote and activate/deactivate inline
- **Per-series prediction editor** (`/admin/series/<id>/predictions`):
  - Table of every user's prediction: `predicted`, `score_made` — each row editable inline via HTMX
  - "Recalculate scores for this series" button — reruns `calculate_score()` for all predictions against current `Series.result`
- **Per-user prediction editor** (`/admin/users/<id>/predictions`):
  - Same view pivoted by user — for disputing individual scores
- `flask scores recalculate` CLI — recomputes all predictions' `score_made` and all users' denormalized `score`
- All write endpoints return HTMX-friendly partial templates (`partials/series_row.html`, `partials/prediction_row.html`)
- `AdminLog` table: every admin write appends a row; last 50 entries visible at `/admin/log`
- **Admin inbox** (`/admin/messages`): list of all messages sent by users to admin, sorted by newest first. Unread count shown as a badge in the admin nav. Admin can reply to a user directly from this page (opens an inline reply form via HTMX). Marking a message read happens automatically on view.
- **Admin → user messaging** (`/admin/messages/send/<user_id>`): admin can open a compose form from the users table and send a direct message to any user. Returns a partial confirmation row via HTMX.

**Exit criterion:** admin can fix any prediction, flip series open/closed, edit a result — all without full page reloads. Admin can read and reply to user messages.

### Phase 4 — Tailwind UI rewrite
- Remove Flask-Bootstrap4 from deps and factory
- Strip all `{% extends "bootstrap/base.html" %}` and `{{ wtf.quick_form() }}` from templates
- Install Tailwind standalone CLI; build `output.css` locally and commit it (no build step on PythonAnywhere)
- `base.html`: sticky header (logo + nav + user menu), flash-message region, `{% block content %}`, footer, HTMX `<script>` tag
- Design tokens: `bg-white` / `dark:bg-neutral-950`, accent `#ED5C2E` as custom Tailwind color `nba`, Inter for body, monospace for scores
- Dark mode: `darkMode: 'class'`; inline `<head>` script reads `localStorage.theme` (or system preference) before paint; toggle button in header
- Pages: home/leaderboard (ranked table, medal icons, season switcher), series list (card grid with status badge + user prediction), series detail (prediction form + all predictions + comments), my predictions, login/register/password reset (centered card), admin (Phase 3), 404, 500
- Mobile-first; test at 375px viewport

**Exit criterion:** no Bootstrap classes remain, Tailwind output CSS under 30 KB gzipped, all pages work on mobile.

### Phase 5 — HTMX integrations
HTMX used only where it clearly pays off. All endpoints return server-rendered partials, never JSON.

- Prediction submit on series card — `hx-post` + swap card body
- Admin inline edits (covered in Phase 3)
- Leaderboard live refresh — `hx-get="/leaderboard/fragment" hx-trigger="every 30s"` during open series; endpoint is rate-limited
- Series open/closed toggle in admin
- Prediction format validation on blur — `hx-post="/validate/prediction"` validates `"X:Y"` format
- Comments — `hx-post` to post, swap comment list partial; admin delete via `hx-delete` + `hx-confirm`
- **User → admin messaging** — users see a "Poruka adminu" (Message to admin) button in their profile or nav. Compose form submits via `hx-post /messages/send`; on success the form swaps to a "Sent" confirmation partial. Enforced server-side: recipient must be an admin, sender cannot be an admin (admin-to-admin blocked).
- **User inbox** (`/messages`) — user sees all messages received from admin, sorted newest first. Unread count badge in the main nav. Messages are marked read on view. Users cannot reply via this inbox — if they want to follow up, they send a new message to admin.

**Exit criterion:** all HTMX targets work; partials in `templates/partials/` are reused in full-page templates. Users can message admins; admins can message users; user-to-user messaging is blocked server-side.

### Phase 6 — Tests + PythonAnywhere deploy
- pytest suite: scoring algorithm (table-driven), auth flows, admin permission boundaries, prediction CRUD, comments, HTMX partial endpoints, messaging (user→admin allowed, admin→user allowed, user→user blocked)
- Ruff: `ruff check` + `ruff format --check` in CI (GitHub Actions on push)
- `wsgi.py` at repo root for PythonAnywhere WSGI configuration
- Static files: Tailwind `output.css` committed; configured via PythonAnywhere dashboard
- SQLite backup: PythonAnywhere Scheduled Task — `sqlite3 /path/to/nba.sqlite ".backup /path/to/backups/nba-$(date +%F).sqlite"` daily, keep last 30 days; document in README

**Exit criterion:** all tests pass, Ruff clean, app live on PythonAnywhere, daily backup scheduled.

## 6. Key Constraints

1. **Never break existing data** — always migrate against a copy of `nba.sqlite` first. The existing rows (users, series, predictions, comments) are valid test fixtures — tests may seed additional data (e.g. test messages) on top of a copy of the real DB, but must never mutate or wipe the production data. The test suite uses a separate in-memory SQLite DB seeded from the same schema; integration tests that need realistic data copy `instance/nba.sqlite` into a temp file at test time.
2. **`calculate_score()` semantics must be preserved exactly** — test table covers all point cases before any refactor
3. **Plain JS only** — no TypeScript, no build pipeline beyond Tailwind CLI
4. **HTMX endpoints return partials only** — never JSON
5. **Serbian UI strings** — keep as default; structure for future `flask-babel` i18n pass (no strings hardcoded in JS)
6. **One blueprint per concern** — `auth`, `main`, `admin` only
7. **All admin writes logged** to `AdminLog`
8. **Closing predictions is manual only** — admin flips `Series.open`; document this prominently in the admin UI

## 7. Confirmed Decisions

- **Email:** Resend SDK, `RESEND_API_KEY` + `MAIL_FROM` env vars, wrapped in `mail.py`
- **Theme:** light/dark toggle persisted in `localStorage`; default to system preference on first visit
- **Leaderboard scope:** per season; `CURRENT_SEASON` env var controls default; season switcher in UI
- **Database:** SQLite in both dev and prod; `instance/nba.sqlite`
- **Deploy:** PythonAnywhere, `wsgi.py`, no Docker
- **Comments:** simple chat-style, no threading/editing/markdown; HTMX post → swap partial; admin can delete
- **Package management:** pip + `requirements.txt`
